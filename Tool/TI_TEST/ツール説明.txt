[概要]
本ツールは、LANタイプの全長保存処理(TI_RPISAVE)の動作確認用ツールになります。
全長保存処理にTCP/IP通信でフレーム画像を送信を行います。
また、接続数については、片面の最大カメラ台数までをサポートします。
処理負荷の分散を行えるようにするために、同一ツールの複数起動をサポートし、
サポート範囲のカメラ台数と開始位置をini設定可能とします。


１．iniファイル設定
①ツール専用のini設定項目を追加
　カメラ台数、カメラ開始位置を設定。

（設定例）　カメラ3,4を担当する場合

#=========================================================
# 全長保存試験ツール
#=========================================================
[TI_TEST]
# カメラ台数
NUM_CAM = 2
# カメラ開始位置(1～)
CAM_START = 3


②判定管理(HT_DEFECT)のini設定項目を設定
　判定管理の全長通信用回線のini設定が必要。

# 全長通信用回線
[HT_DEFECT_RPI_TO_0_0]
SAVE_IP		= 127.0.0.1
SAVE_PORT	= 11000
[HT_DEFECT_RPI_TO_0_1]
SAVE_IP		= 127.0.0.1
SAVE_PORT	= 11001
　　・
　　・　（省略）
　　・



２．複数起動時のメール連携

複数起動時に、実行タイミングの同期を図るため、
以下のボタン押下時は、メールスロット通知を行います。

　①周期実行
　　周期実行のボタンON/OFFに連動して、他のツールへの起動を行います。

　②フレームNo
　　フレームNoの設定を行うことで、次フレームNoを他のツールに通知し、フレームNoを合わせます。


３．初期値
　試験の効率化のため、下記の項目は初期値を入れてます。
　プログラム内のハードコーディングのため、変更する場合はプログラム修正が必要です。

　・[BMP]　ファイルパス、周期、開始No、終了No
　・[連続稼働]　フレームNo、連続稼働ボタンON


４．起動時動作
　ツール起動後に10秒後に自動で検査開始します。
　全長保存処理は、自動では検査開始しないため、検査開始が必要となります。
　検査停止している状態では、回線接続していても、全長保存はされないため、試験時は状態確認が必要！！
